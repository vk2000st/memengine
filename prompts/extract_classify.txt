## Task
Analyze the conversation and extract memory candidates. Focus only on information that:
- Is factual, stable, or reflects the user's preferences, goals, relationships, or important events
- Would be useful to know in future conversations with this user
- Includes state changes and life updates (moving cities, changing jobs, switching preferences)
- Is not ephemeral small talk or temporary state
- Do NOT skip messages because they use casual language like "btw", "actually", "you know" — if the underlying content is a personal fact, extract it

Do NOT extract:
- Greetings, pleasantries, or filler content
- Questions the user asked (unless they reveal a preference or goal)
- Obvious or universally known facts
- Content that is too vague to be actionable
- Random characters, gibberish, or messages with no intelligible meaning

## Memory Types
- fact: A factual piece of information (name, location, job)
- preference: Something the user likes, dislikes, or prefers
- goal: Something the user is working toward
- skill: Something the user knows or is learning
- context: Current situational info (plan tier, active ticket)
- event: A significant life event or experience
- constraint: A limitation or requirement (allergy, disability)

## Importance Score Guidelines (0.0 to 1.0)
- 0.9-1.0: Critical personal information (medical, core identity, life goals)
- 0.7-0.8: High-value preferences and frequently relevant facts
- 0.5-0.6: Useful context that improves interactions
- 0.3-0.4: Mildly useful, situational context
- Below 0.3: Not worth storing
