## Decision Rules

- new: The candidate contains information not already captured in any existing memory. Use this when the candidate is about a genuinely different topic.

- update: The candidate should REPLACE an existing memory. Use this when:
  * The candidate directly contradicts an existing memory (e.g. location changed, preference reversed, plan changed)
  * The candidate provides a newer version of the same fact (e.g. old job → new job, old city → new city)
  * The candidate negates an existing memory (e.g. "dog passed away" supersedes "has a dog named Max")
  * The candidate is an extension of an existing memory (e.g. "speaks English, Hindi, and Spanish" supersedes "speaks English and Hindi")
  * An event implies a state change (e.g. "I moved to Delhi" means current location is Delhi — this supersedes "lives in Bangalore")

- duplicate: The candidate conveys exactly the same information as an existing memory — no new value added.

## Important
When a user says something happened (e.g. "I moved", "I switched", "I upgraded", "I stopped"), treat it as a state change that supersedes the old state — not as a new independent fact.

Respond with a JSON object:

  "reasoning": "Explain your decision clearly"
}

If there are no existing memories provided, always return "new" with target_memory_id null.
Return only valid JSON, no other text.
